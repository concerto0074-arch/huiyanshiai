# 登录功能使用说明

## 问题排查

### 1. 无法登录的可能原因

#### 后端服务器未启动
- **症状**：登录时显示"登录失败：Failed to fetch"或"网络错误"
- **解决方案**：
  1. 打开命令行工具
  2. 进入`api`目录：`cd api`
  3. 启动后端服务器：`python start_server.py`
  4. 确保服务器在 http://127.0.0.1:5000 上运行

#### 数据库未初始化
- **症状**：登录时显示"登录失败：用户名或密码错误"
- **解决方案**：
  1. 确保后端服务器已启动
  2. 运行数据库初始化脚本：`python init_db.py`
  3. 运行管理员创建脚本：`python init_admin.py`

#### 用户名或密码错误
- **默认测试账号**：
  - 普通用户：`testuser` / `test123`
  - 管理员：`admin` / `admin123`

### 2. 调试方法

#### 前端调试
1. 打开浏览器开发者工具（按F12）
2. 切换到"控制台"（Console）选项卡
3. 尝试登录，查看控制台输出的调试信息

#### 后端调试
1. 确保后端服务器在调试模式下运行
2. 查看命令行中的服务器日志
3. 检查数据库中的用户信息

### 3. 手动测试API

可以使用Postman或curl测试登录API：

```bash
# 使用curl测试登录API
curl -X POST http://127.0.0.1:5000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"test123"}'
```

## 功能说明

### 普通用户登录
1. 点击页面右上角的"用户中心"按钮
2. 选择"普通用户登录"选项卡
3. 输入用户名和密码
4. 点击"登录"按钮

### 管理员登录
1. 点击页面右上角的"用户中心"按钮
2. 选择"管理员登录"选项卡
3. 输入管理员用户名和密码
4. 点击"登录"按钮
5. 登录成功后会跳转到管理员后台页面

## 技术支持

如果您仍然无法解决登录问题，请检查：
1. Python环境是否正确安装
2. 必要的依赖是否已安装：`pip install -r requirements.txt`
3. `.env` 中的 `SUPABASE_URL`、`SUPABASE_ANON_KEY`、`JWT_SECRET` 是否已正确配置
4. 防火墙是否阻止了5000端口的访问
