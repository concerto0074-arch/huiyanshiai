## 3. RPCA程序实现分析

### 3.1 程序结构

**核心功能**：将基因表达数据分解为低秩（clean data）和稀疏（noise data）成分

**主要模块**：
1. 动态路径处理（第2-8行）
2. 数据读取与预处理（第9-13行）
3. RPCA分解计算（第16-21行）
4. 结果提取与后处理（第24-31行）
5. 结果输出（第33-36行）

### 3.2 技术亮点

#### 动态路径管理
```r
# 获取脚本所在目录（解决相对路径问题）
args <- commandArgs(trailingOnly = FALSE)
script_path <- args[grep("^--file=", args)]
script_path <- sub("^--file=", "", script_path)
script_dir <- dirname(script_path)
```

**优势**：确保在任何目录下运行程序都能正确访问输入/输出文件

#### RPCA算法实现
```r
res <- rpca(lungdata1, 
           lambda = 1/sqrt(max(dim(lungdata1))), 
           mu = prod(dim(lungdata1))/(4*sum(abs(lungdata1))),
           term.delta = 10^(-7), max.iter = 10000)
```

**参数说明**：
- `lambda`：稀疏成分正则化参数（Candes默认值）
- `mu`：算法步长参数
- `term.delta`：迭代终止阈值（10⁻⁷）
- `max.iter`：最大迭代次数（10,000次）

### 3.3 结果输出

程序生成两个核心输出文件：

| 文件名 | 路径 | 内容 |
|--------|------|------|
| cleandata.csv | 程序目录 | 低秩成分（数据主要结构/生物信号） |
| noisedata.csv | 程序目录 | 稀疏成分（噪声/异常值/干扰信号） |

**输出格式**：保留原始基因ID，便于后续分析
