# 慧眼识癌 AI 辅助系统 - 项目架构说明

## 📋 项目概述

慧眼识癌是一个基于机器学习的癌症风险预测平台,整合了多维度健康数据分析、高维基因表达数据处理(R算法集成)、用户认证系统和SaaS服务架构。

## 📁 目录结构详解

### 1. 后端 (Backend)

```
backend/
├── app.py                      # 🔥 主应用入口 (Flask服务器)
├── config.py                   # ⚙️ 配置文件 (模型路径、特征定义等)
├── utils.py                    # 🛠️ 工具函数 (日志、数据加载、模型加载)
├── r_integration_service.py    # 🧬 R算法集成服务 (处理高维基因数据)
├── requirements.txt            # 📦 Python依赖
├── models/                     # 🤖 训练好的机器学习模型
│   ├── random_forest_model.joblib
│   ├── svm_model.joblib
│   ├── logistic_regression_model.joblib
│   └── training_results.joblib
└── data/                       # 📊 训练数据
    └── cancer_data.csv
```

#### 核心文件说明

**app.py** - 主后端服务
- Flask应用初始化
- 所有API路由定义
- 用户认证逻辑 (JWT + bcrypt)
- 机器学习预测接口
- R算法调用接口
- 前端静态文件服务

**config.py** - 配置管理
- 模型路径配置
- 特征定义 (年龄、性别、BMI、吸烟史等)
- 算法超参数
- API服务器配置

**r_integration_service.py** - R算法集成
- 调用R脚本处理高维基因数据
- 支持多种算法: MSGL、AMRSOGL、RAMRSGL等
- 临时文件管理

## 🔐 认证流程

### 1. 用户登录流程
```
用户输入账号密码
    ↓
前端发送 POST /api/login
    ↓
后端验证用户名密码 (bcrypt校验)
    ↓
生成JWT Token (有效期24小时)
    ↓
返回Token + 用户信息
    ↓
前端存储Token到localStorage
    ↓
后续请求携带 Authorization: Bearer {token}
```

### 2. Token验证流程
```
前端发起API请求
    ↓
携带 Authorization Header
    ↓
后端 @token_required 装饰器验证
    ↓
解析JWT获取用户信息
    ↓
注入 request.user
    ↓
执行业务逻辑
```

## 🤖 机器学习模型

### 已训练模型

1. **随机森林 (Random Forest)**
   - 文件: `random_forest_model.joblib`
   - 特点: 集成学习,抗过拟合
   - 适用: 通用场景

2. **支持向量机 (SVM)**
   - 文件: `svm_model.joblib`
   - 特点: 小样本高维数据
   - 适用: 精确分类

3. **逻辑回归 (Logistic Regression)**
   - 文件: `logistic_regression_model.joblib`
   - 特点: 计算快速,易解释
   - 适用: 快速评估

### 输入特征 (7维)
- age: 年龄
- gender: 性别 (0=男, 1=女)
- bmi: BMI指数
- smoking: 吸烟史 (0=从不, 1=曾经, 2=当前)
- tumor_marker1: 肿瘤标志物1 (如CEA)
- tumor_marker2: 肿瘤标志物2 (如WBC)
- mutation: 基因突变 (0=阴性, 1=阳性)

### 输出结果
- 风险等级: 0=低风险, 1=中风险, 2=高风险
- 概率分布: 三个类别的概率值

## 🧬 R算法集成

### 支持的算法

1. **MSGL算法** - 多任务稀疏群Lasso
2. **AMRSOGL算法** - 自适应多响应稀疏重叠群Lasso
3. **RAMRSGL算法** - 鲁棒自适应多响应稀疏群Lasso
4. **AI智能匹配** - 自动选择最优算法

### 数据要求
- 格式: CSV文件
- 内容: 高维基因表达数据
- 示例: `frontend/sample_gene_data.csv`

### 处理流程
```
上传CSV文件
    ↓
保存到临时目录 (tmp_data/)
    ↓
调用对应R脚本
    ↓
解析R输出结果
    ↓
返回风险评估
    ↓
清理临时文件
```

## 🗄️ 数据库设计 (Supabase)

### users 表
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,  -- bcrypt哈希
    phone VARCHAR(20),
    age INTEGER,
    gender VARCHAR(10),
    role VARCHAR(20) DEFAULT 'user',  -- user | admin
    status VARCHAR(20) DEFAULT 'active',  -- active | banned | inactive
    plan VARCHAR(20) DEFAULT 'free',  -- free | standard | premium
    plan_expire_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

## 🐛 常见问题

### 1. 模型加载失败
- 检查 `backend/models/` 目录是否存在模型文件
- 运行 `backend/train.py` 重新训练模型

### 2. Supabase连接失败
- 检查 `.env` 中的配置是否正确
- 访问 `/api/health` 查看连接状态

### 3. R脚本执行失败
- 确保安装了R环境
- 检查 `程序/` 目录下的R脚本路径

### 4. CORS错误
- 检查 `backend/app.py` 中的CORS配置
- 确保前端URL在白名单中
